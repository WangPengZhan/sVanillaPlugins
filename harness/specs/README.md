# Specs

Use this section for shared behavior specs and design decisions that should be
visible to multiple AI agents.

Good entries include:

- behavior contracts;
- API or data model decisions;
- migration plans;
- cross-plugin design notes;
- rejected alternatives when the reasoning should be preserved.

Tool-specific task notes can live outside this tree when needed. Mirror or
summarize only the parts that should become shared long-term context.

## Current Files

- `plugin-architecture.md`: shared plugin architecture and ABI contract.
- `sdd-and-harness-contract.md`: SDD plus harness engineering contract.
