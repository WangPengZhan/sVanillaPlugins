# Knowledge

Use this section for durable facts about the repository, domain, dependencies,
and host integration.

Good entries include:

- architecture summaries;
- plugin API facts;
- platform behavior notes;
- dependency and toolchain constraints;
- links to canonical local documentation.

Prefer factual notes over decisions. If a note explains a decision or behavior
contract, put it in `../specs/` or link to the relevant spec.

## Current Files

- `project-overview.md`: current repository architecture and canonical docs.
- `plugin-inventory.md`: current plugin list and URL support summary.
