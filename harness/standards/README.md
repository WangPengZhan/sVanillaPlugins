# Standards

Use this section for engineering standards that apply across agents and tasks.

Good entries include:

- coding conventions;
- test expectations;
- documentation rules;
- security and privacy constraints;
- review checklists;
- repository hygiene rules.

Keep standards concrete and enforceable. If a rule has exceptions, describe the
conditions for the exception.

## Current Files

- `engineering-standards.md`: current coding, build, dependency, testing, and
  repository hygiene standards.
